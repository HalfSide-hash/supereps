Introduction

Materia is fully responsive admin web app built with AngularJS, Bootstrap 3.3 and LESS stylesheet. Most of the jquery plugins is compatible with angularjs so it does work.

File Structure

Within the download you'll find the following file structure and contents

Once downloaded, unzip the compressed folder and you'll see something like this:

Bower

NOTE: You don't need bower anyway because all the scripts and css files already included with src folder.

As most of the plugins's css files must be edited before used by template, so it's not possible to directly reference it from bower_components folder. This means you don't need to run bower install command to work with development version (src). There's also bower.json file included to see which scripts are used by the template.

Less

Materia uses less to generate css files. The css files generation has been simplified by grunt task runner (read later). Also, we use less version of bootstrap (3.3.3) so variables.less file of bootsrap is shared among all the stylesheets and it's modified. So, if newer version is available for upgrade, then simply copy the bootstrap less folder and do the following:

  • Move the variables.less file one step back to src/styles.
  • Open the bootstrap.less file and change the above file path to point to the new path

The main less file is main.less. It includes all the less files except bootstrap folder. On output, less compiler generate two css files in dist folder -- main.min.css and boostrap.min.css.

It's probably a bit complex to you if it's your first time using Grunt. However, you'll fail in love with it once you get it, it saves you from repetitive tasks. Project like jQuery, Bootstrap and many more use and love GruntJS.

Important grunt tasks commands

  • grunt build
    will build the production version of template in dist folder.

  • grunt dev
    will compile less file into css (bootstrap.min.css and main.min.css) in src/_tmp folder.

  • grunt cleanBuild
    will clean the dist folder.

How to Run It?

Simply upload the dist folder into the server and run it. However if you want to test it locally, then do the following:

  • Install a server locally. You can use any of them (WAMP, MAMP, nodejs *http-server*). Here we are using nodejs http-server
  • Then execute this command http-server from the command line.
  • It'll return something like this:
    Starting up http-server, serving ./ on port: 8080
  • Then you can visit the template on 
    http://localhost:8080
    within your browser.

Custom Angular Directive

There are some custom directive written for this template. The options provided you'll find in the file. Here's I'll provide the list:

  • ui-range-slider: For generating custom bootstrap range. Used by boostrap-slider.js plugin.
  • collapse-nav-accordion: For collapsing child elements of sidebar nav.
  • toggle-off-canvas: Toggle Off-canvas sidebar nav in mobile device.
  • highlight-active: For highlighting active nav.
  • ui-checkbox: For custom checkbox and radio.
  • custom-scrollbar: For custom scrollbar. Used with perfect-scrollbar.js plugin.
  • custom-page: Used by custom pages in full body mode (no sidebar and header).
  • sparkline: Quickly generate inline charts.
  • angular-c3: C3 chart directive. It's an element based directive.
  • ui-range-slider: Create range slider.

Custom Angular Directive

In order to do development with it, you'll need have the following tools installed:

  • Node.js (NPM comes along with it)
  • Grunt (npm package) for task automation
  • Bower (npm package) for dependency (optional). I've already included all the necessary scripts from bower in src/scripts directory.
  • lesscss (npm package) for compiling less to css

    • And you'll need to know how to use them, click the links to learn more if you not familiar with these yet. If it's your first time using these tools, it's probably too complex or even overwhelming to you. You'd better find someone to help you.

    If you already know how to use them, then just do a few things to start developing:

    1. Install all the above npm packages and nodejs
    2. Start the http-server inside the src folder using the commandhttp-server.

    All the files for development are under Materia/src/ folder.

    Basic Template Structure

    The basic structure of template (index.html) is quite simple. It contains 2 parts: The header, which load all the stylesheets. The body, which contains the content of App and load all the scripts.

    The Header

<head>
 <meta charset="utf-8">
 <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
 <meta name="description" content="Materia - Admin Template with Angularjs">
 <meta name="keywords" content="minty, webapp, admin, dashboard, template, ui">
 <meta name="author" content="solutionportal">
 <!-- <base href="/"> -->

 <title>Materia - Admin Template</title>

 <!-- Icons -->
 <link rel="stylesheet" href="fonts/ionicons/css/ionicons.min.css">
 <link rel="stylesheet" href="fonts/font-awesome/css/font-awesome.min.css">

 
 <!-- Css/Less Stylesheets -->
 <link rel="stylesheet" href="styles/angular-material.min.css">
 <link rel="stylesheet" href="styles/vendors/bootstrap.min.css">
 <link rel="stylesheet/less" href="styles/main.less">

</head>
